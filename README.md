# Martyn Variant Effects Evaluator (GAME)

A [GAME](https://github.com/de-Boer-Lab) Evaluator that
benchmarks how well a Predictor recovers **measured regulatory-variant effects on gene expression**
at the **PPIF** and **IL2RA** loci, in **THP-1 monocytes** and **Jurkat T cells**.

Measured effects come from the Variant-EFFECTS screens of Martyn et al., *Rewriting regulatory
DNA to dissect and reprogram gene expression*, **Cell** 188(12):3349–3366 (2025)
— doi:[10.1016/j.cell.2025.03.034](https://doi.org/10.1016/j.cell.2025.03.034).
The Evaluator sends one prediction request per cell type to a Predictor, then reports the
**Pearson correlation** between predicted and measured variant effects (log2fold-change).

## Quick start

Download the container image and evaluator_data from Hugging Face (see [Important Links](#important-links) above).

```bash
apptainer run \
  -B /abs/path/to/evaluator_data:/evaluator_data \
  -B /abs/path/to/predictions:/predictions \
  martyn_evaluator.sif PREDICTOR_HOST PREDICTOR_PORT /predictions
```

| Argument | Meaning |
|----------|---------|
| `PREDICTOR_HOST` | IP or hostname of the running Predictor |
| `PREDICTOR_PORT` | Port the Predictor is listening on |
| `/predictions`   | Mounted output dir for predictions + summary CSV |

`config.py` auto-detects the container (via `/.singularity.d`) and reads inputs from
`/evaluator_data`. The mount **must** contain `gene_reference_sequences.csv` and, for each cell
type, `THP1/variant_sequences.csv`, `THP1/all_THP1.tsv`, `Jurkat/variant_sequences.csv`,
`Jurkat/all_Jurkat.tsv`. See [`evaluator_data/README.md`](martyn-variant-effects-evaluator/evaluator_data/README.md).

## What it does

For each cell type (THP-1, Jurkat):

1. **Build the request** from sequence files (`variant_sequences.csv`) in the corresponding cell type folders and the reference promoter sequence (`gene_reference_sequences.csv`).
2. **Negotiate serialization formats** with the Predictor (`GET /formats`) and **POST** the request
   to `/predict`.
3. **Save** the raw predictions and, on success, **compute Pearson r** between the predicted and
   measured per-variant log2 fold-change.

The variant/reference sequences themselves were generated by
[`evaluator_data/parse_Engreitz_data.py`](martyn-variant-effects-evaluator/evaluator_data/README.md), details are documented in the `evaluator_data/` README.

```

variant_sequences.csv + gene_reference_sequences.csv ──► data_loader ──► GAME /predict request ──► Predictor
                                                                        │
Predictor response ──► saved predictions .json                         │
                   └─► metrics: predicted log2FC vs measured log2FC ──► evaluation_summary_*.csv
```

## Inputs the container consumes

| File | Read by | Format | Required columns |
|------|---------|--------|------------------|
| `gene_reference_sequences.csv` | `data_loader` | comma-sep | `gene`, `promoter_length`, `strand`, `sequence` |
| `{cell}/variant_sequences.csv` | `data_loader` | tab-sep | `VariantID_h19`, `gene_symbol`, `variant_sequence` |
| `{cell}/all_{cell}.tsv` | metrics | tab-sep | `VariantID_h19`, `gene_symbol`, `log2_fold_change` |

Cell-type → folder mapping is hard-coded in `evaluator_RestAPI.py`:
`THP-1 monocytes → THP1`, `Jurkat T cells → Jurkat`.

## Outputs (written to the mounted output dir)

- **Raw predictions**, one JSON per cell type:
  `{evaluator_name}_predictions_{cell}_from_{predictor_name}.json`
- **Summary metrics**, a single tab-separated file appended across runs/cell types:
  `evaluation_summary_{evaluator_name}.csv`, with columns
  `evaluator_name, description, predictor_name, time_stamp, metric, value, prediction_task(s)_data`.

## The GAME request

For each cell type, `data_loader.create_json_from_tsv()` de-duplicates on `VariantID_h19` and emits:

```jsonc
{
  "readout": "point",
  "prediction_tasks": [
    {
      "name": "martyn_<cell_type>",
      "type": "expression",
      "cell_type": "<cell_type>",
      "scale": "log",
      "species": "homo_sapiens"
    }
  ],
  "sequences": {
    "PPIF": "<2 Mb reference sequence>",           // + "IL2RA" for Jurkat
    "chr10:...:REF>ALT": "<2 Mb variant sequence>",
    ...
  },
  "prediction_ranges": {
    "PPIF": [1000000, 1000000 + promoter_length],    
    "chr10:...:REF>ALT": [1000000, 1000000 + promoter_length], 
    ...
  }
}
```

Each variant sequence is 2 Mb (±1 Mb around the promoter start), so the prediction range points at
the promoter window near the center of the sequence. Reference sequences are keyed by gene symbol
(`PPIF`, `IL2RA`); variant sequences by `VariantID_h19`. 

The Predictor is expected to return, per task, a `predictions` dict mapping each sequence ID to a
scalar.

## The metric

`evaluator_metrics_calculator.calculate_pearson_r()`:

1. For each variant, compute a **predicted log2 fold-change** relative to its gene's reference
   prediction:
   - `scale == "log"` (requested): `log2FC = pred_variant − pred_reference`.
   - `scale == "linear"` (fallback if the Predictor reports linear): `log2FC = log2(variant / reference)`,
     with non-positive values dropped. Pearson r is invariant to log base, so a linear-returning
     Predictor is still scored correctly rather than rejected.
2. Merge with measured `log2_fold_change` on `VariantID_h19` and compute **Pearson r**.

Return-value semantics (deliberately distinct):

| Return | Meaning | Reported as |
|--------|---------|-------------|
| `float` | Correlation computed | the value |
| `0.0` | Ran but useless — zero variance in predicted or measured log2FC, or a NaN r | `0.0` |
| `None` | Disqualified — error payload, NA predictions, wrong scale, or < 2 paired points | `"NaN"` |

Metrics are only computed when the Predictor returns `200 OK` **and** every task returned one
prediction per sequence.

## Configuration (`config.py`)

| Setting | Default | Notes |
|---------|---------|-------|
| `REQUEST_FORMAT` | `application/json` | Downgraded to JSON if the Predictor can't accept it |
| `RESPONSE_FORMAT` | `application/msgpack` | Downgraded to JSON if unsupported |
| `MAX_RETRIES` | 50 | Network-error retries only (HTTP 4xx/5xx are not retried) |
| `RETRY_INTERVAL` | 30 s | Constant wait between retries |
| `EVALUATOR_DATA_DIR` | `/evaluator_data` (container) / `./evaluator_data` (dev) | Auto-detected |
